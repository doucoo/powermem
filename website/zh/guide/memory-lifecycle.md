# 记忆生命周期

PowerMem 管理记忆的完整生命周期——它们不是静态笔记堆。

## 1. 提取

当输入到达时（对话、行为、反馈），LLM 提取结构化记忆：

```
输入: "我刚把 auth 服务部署到 prod，结果崩了，
       因为数据库迁移没有先执行。"

提取:
  - 事实: "Auth 服务已部署到生产环境"
  - 经验: "不先执行 DB 迁移就部署 auth 服务会导致崩溃"
  - 实体: "auth 服务"（类型: service）
```

## 2. 更新与合并

新记忆与现有记忆对比，相似记忆合并：

```
现有: "用户偏好暗色模式"
新增: "用户喜欢 IDE 使用暗色主题"
合并: "用户偏好所有 IDE 使用暗色模式/主题"
```

防止记忆膨胀并保持信息聚合。

## 3. 强化

每次记忆被成功检索使用时，其**强度（S）**增加：

```python
S_new = S_old + reinforcement_factor  # 默认: 每次检索 +0.1
```

更强的记忆衰减更慢，在搜索结果中排名更高。

## 4. 衰减（艾宾浩斯遗忘曲线）

未使用的记忆随时间衰减：

```
R(t) = e^(-t/S)
```

| 距上次访问时间 | S=1.0 | S=2.0 | S=5.0 |
|----------------|-------|-------|-------|
| 1 小时         | 37%   | 61%   | 82%   |
| 1 天           | ~0%   | 30%   | 67%   |
| 1 周           | ~0%   | ~0%   | 45%   |

低于可检索性阈值的记忆被**归档**（非删除）。当上下文需要时可重新激活。

## 5. 重新激活

当归档记忆再次变得相关时：
- 重新计算可检索性
- 部分恢复强度
- 重新进入活跃记忆池

这防止了永久丢失，同时保持活跃集干净。
