# 架构说明

PowerMem 采用 5 层架构：

```
┌─────────────────────────────────────────────────┐
│  外部层                                          │
│  Claude Code │ Cursor │ VS Code │ MCP │ CLI     │
├─────────────────────────────────────────────────┤
│  API 层                                          │
│  Python SDK │ HTTP REST │ MCP Server │ CLI      │
├─────────────────────────────────────────────────┤
│  核心层                                          │
│  记忆管线 │ 检索引擎 │ 用户画像                   │
│  衰减引擎 │ 蒸馏系统 │ 子存储                     │
├─────────────────────────────────────────────────┤
│  模型层                                          │
│  LLM 提供商 │ 嵌入模型提供商 │ Re-rank           │
├─────────────────────────────────────────────────┤
│  存储层                                          │
│  OceanBase │ PostgreSQL │ SQLite │ seekdb        │
└─────────────────────────────────────────────────┘
```

## 记忆管线

当对话或行为发生时：

1. **输入** — 文本/图片/音频通过 SDK、HTTP 或 MCP 到达
2. **LLM 提取** — 模型识别关键事实、决策和实体
3. **更新与合并** — 新事实与现有记忆对比，重复项合并
4. **嵌入** — 记忆文本转换为向量
5. **存储** — 记忆 + 向量 + 元数据持久化
6. **图谱更新** — 提取实体关系并存入图谱

## 检索引擎

查询通过三条并行路径处理：

```
查询 → [向量检索] ──┐
    → [全文 BM25] ──┤→ RRF 融合 → Re-rank → Top-K 结果
    → [图谱检索] ──┘
```

### 互惠排名融合（RRF）

每条检索路径返回排名列表，RRF 合并：

```
score(d) = Σ 1 / (k + rank_i(d))
```

其中 `k` 为平滑常数（默认 60），`rank_i(d)` 为文档 `d` 在路径 `i` 中的排名。

## 衰减引擎

实现艾宾浩斯风格遗忘：

```
R(t) = e^(-t/S)
```

- `R` = 可检索性（回忆概率）
- `t` = 距上次访问的时间
- `S` = 记忆强度（每次成功检索时增加）

低于阈值的记忆被归档但不删除——当上下文需要时可重新激活。

## 子存储与路由

记忆按领域组织到子存储中：

- **事实** — 静态知识（用户偏好、项目事实）
- **经验** — 情景记忆（发生了什么、结果如何）
- **技能** — 程序性知识（如何做事情）
- **自定义** — 用户定义的特定领域子存储

查询基于意图分类自动路由到相关子存储。

## Web 面板

内置面板提供：

- 按用户、Agent、类型分布的记忆计数
- 健康指标（衰减曲线、检索延迟）
- 搜索界面
- 记忆增删改查操作
